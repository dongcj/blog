1.4.2. ReadAhead

About ReadAhead: this feature will read more stuff and store in the cache, guessing the system may access it soon. We’re going to enable an enhanced version of readahead: the adaptative one. With this option, readahead will only be enabled if the controller receive several access to sequencial sectors. If not, it won’t be used to avoid filling cache with randon useless data (in case of randomly accessed sector).

1.4.3. write cache

Enable write cache，data will be lost! Write cache should be enabled ONLY if you have a battery pack on your controller！

# 按照上面的 /dev/sda 的分区例子依次给剩下的 5 块硬盘 sdc, sdd, sde, sdf, sdg 分区

...
$ fdisk /dev/sdf
...
$ fdisk /dev/sdg

2.3. 创建 RAID

# 在上面的 6 个相同大小的分区上创建 raid10
$ mdadm --create /dev/md0 -v --raid-devices=6 --level=raid10 /dev/sda1 /dev/sdc1 /dev/sdd1 /dev/sde1 /dev/sdf1 /dev/sdg1

2.4. 给 md0 设备创建分区和文件系统

$ fdisk /dev/md0
$ mkfs.ext4 /dev/md0p1

$ mkdir /raid10
$ mount /dev/md0p1 /raid10

2.5. 修改 /etc/fstab 启动时自动挂载

在上面的 /etc/fstab 文件里使用 /dev/md0p1 设备名不是一个好办法，因为 udev 的缘故，这个设备名常在重启系统后变化，所以最好用 UUID，使用 blkid 命令找到相应分区的 UUID
