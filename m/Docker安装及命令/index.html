Ubuntu安装docker

先升级kernel及安装linux-image-extra-*的包(为了支持bbr、aufs)

# 防止出现第6步的错误：WARNING: No swap limit support
vi /etc/default/grub
  GRUB_CMDLINE_LINUX_DEFAULT="cgroup_enable=memory swapaccount=1"

update-grub

安装apt cert

apt-get update
apt-get install -y apt-transport-https  ca-certificates  curl  software-properties-common

install the apt key

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
# Verify the key ID
apt-key fingerprint 0EBFCD88

install the apt repository

add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu zesty stable"

install docker-engine

apt-get update
# 查看可用的docker版本
apt-cache madison docker-engine

# 安装指定的版本
apt-get -y install docker-engine[=<VERSION_STRING>]

或者直接使用以下方法安装

检查

docker info | grep WARN

WARNING: No swap limit support 解决方法见第1步，配置 grub 参数

WARNING: bridge-nf-call-ip6tables is disabled 解决方法：

vim /etc/sysctl.conf
    net.bridge.bridge-nf-call-ip6tables = 1
    net.bridge.bridge-nf-call-iptables = 1
    net.bridge.bridge-nf-call-arptables = 1
    sysctl -p

CentOS安装docker

(本方法只适用于 centos6 及以下)

yum -y install http://dl.fedoraproject.org/pub/epel/6/x86_64/epel-release-6-8.noarch.rpm
yum install docker-io

(本方法只适用于 centos7 及以上)

docker 直接使用外部块存储

映射为 container 内部盘，可以自定义使用read、write、mknode操作

docker run --device=/dev/sdc:/dev/xvdc:[rwm] --device=/dev/sdd --device=/dev/zero:/dev/nulo -i -t ubuntu ls -l /dev/{xvdc,sdd,nulo}

Restart policies

docker run --restart=[always|no|on-failure|unless-stopped] redis

向容器中增加主机名与IP对应(add /etc/hosts entry)

docker run --add-host=docker:10.180.0.1 --rm -it debian

设置容器的ulimit

docker run --ulimit nofile=1024:1024 --rm debian sh -c "ulimit -n"

docker重启daemon不重启container, 以下二种方法任一种都可以

1. 将 /etc/docker/daemon.json 中的 "live-restore" 设置为 true，然后SIGHUP
2. sudo dockerd --live-restore

Docker Daemon的配置文件

Docker小技巧

set metadata on container

直接设置sysctl(不能与–network=host同用)

docker run --sysctl net.ipv4.ip_forward=1 ubuntu

从container的变化中新建一个image

docker commit $CONTAINER_ID [REPOSITORY[:TAG]]

docker传递变量进系统, 类似从kernel传递至env效果一样

dock run <IMAGEID>:TAG --env-file=xx.yaml

常用Docker命令

备份容器中的数据（将容器中的数据目录拷贝至当前目录下）

docker run --rm --volume-from dbdata -v ${pwd}:/backup  ubuntu tar cvf /backup/backup.tar /dbdata

Docker将本地目录映射给container

docker run -it --rm -v /mnt/Docker:/usr/Download  matthewdailey/ubuntu-openbr:v1 /bin/bash

/mnt/Docker为宿主机目录:/usr/Download为container里的目录(Download不存在也可)

如果仅使用-v选项，则可以将目录share出来，供其它container共享（使用–volumes-from=SHARED_CONTAINER_ID）

iptables -t nat -A  DOCKER -p tcp --dport 3306 -j DNAT --to-destination 172.17.0.2:3306

docker image step by step

目录结构

目录大体分为两个：一个是blobs，一个是repositories。blobs中主要存放数据文件，可以看出都是经过sha256计算后的ID。repositories目录中放镜像的描述信息，记录了一个镜像有哪些layer，tag对应的manifest文件，link文件是一个文本文件，内容是一个形如“sha256:cf34a09a90b54c…”的64位ID，这个ID对应在blob中的文件其实就是这个image的manifest文件。

manifest文件

manifest文件描述了一个镜像的元信息，包括了layer的数据ID，layer的配置等，文件格式是json形式的文本文件。
